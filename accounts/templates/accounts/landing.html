Capabilities

Everything you need to sign at scale

From a single REST call to a DocuSign-style portal — one platform, one audit log, your DSC certificate.

REST API signing

POST a PDF with your API key. Use inline PFX or a saved certificate alias. Automatic anchor detection and signature placement.

In-browser signing

Upload a PDF in the portal, preview placement with PDF.js, sign with a saved certificate, and download — no integration required.

Saved certificates

Store PFX files encrypted per tenant. Reference certificates by alias in API calls or the portal — passwords are never persisted.

Signing audit trail

Every signature records SHA-256 hashes before and after, detected document type, client IP, and whether it came from the API or browser.

Signature styling

Opt in to custom anchor text, box size, and placement offsets per tenant. Existing API clients keep platform defaults until you enable it.

Tenant dashboard

Organization accounts with API keys, monthly quotas, activity history, and certificate management — ready for production workloads.

Two ways to sign

Integrate in minutes, or sign from the portal

Choose the path that fits your team. Both flows share the same certificates, quotas, and audit log.

For developers

API workflow

  1. Register your organization and get approved
  2. Upload a PFX and create an API key
  3. POST /api/signpdf-pfx with your PDF and certificate
  4. Receive the signed PDF plus signing ID and hash prefixes

Browser workflow

  2. Upload a saved certificate and open Sign PDF

  4. Sign and download — event logged in activity

Trust & compliance

Built for business-critical documents

IG E-Sign uses your organization's DSC/PFX certificate for legally valid digital signatures. Platform defaults locate the Authorised Signatory anchor in your PDFs. Every signing event is traceable in your dashboard.

  • Rate limiting and monthly signing quotas
  • Document type auto-detection from PDF content
  • Incite Gravity — ISO 27001:2022 certified
