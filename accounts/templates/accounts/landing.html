Class 3 USB DSC

Sign with the token — not a file upload

For eMudhra, Capricorn, and other Class 3 DSC tokens, the private key never leaves the USB device. IG E-Sign prepares the PDF in the cloud; signing happens locally via PKCS#11 on the machine where the token is plugged in.

  1. Prepare Your backend or portal creates a sign job with POST /api/sign/usb/ and the target device_id.
  2. Sign locally The IG E-Sign Agent on Windows prompts for the token PIN and signs via PKCS#11.
  3. Retrieve Poll job status, then download the signed PDF - same audit trail as PFX signing.
  • One-time agent pairing from the portal
  • Per-job sign_token for secure local triggers
  • Works for ERP automation and portal upload flows
  • Signed PDF verified server-side before delivery

USB DSC workflow
